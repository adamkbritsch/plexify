Telegram source … {% if get_config('telegram_enabled','0') == '1' %}ON{% else %}OFF{% endif %} (?)

Drives @BeatSpotBot (FLAC) as a fallback when Soulseek and SpotiFLAC both come up empty. It signs in with a one-time session string generated from your own Telegram API credentials — your phone number and login code never leave your NAS.

How to generate the session string
  1. Get an API ID + API hash from my.telegram.org → API development tools. Save them above (or just have them ready).
  2. From the NAS, run this once and follow the phone + login-code prompts: 
    docker compose exec {{ container_name }} python -m app.tg_login
  3. It logs in as you and saves the session automatically — it also prints the string if you'd rather paste it here yourself. Your phone/code/2FA are never stored.
  4. Tick Enable, Save, then click Test session — it should show your account and that @BeatSpotBot is reachable.

Always on, for every source: failing sources cool down and fail over in try-order; the Qobuz mirror is benched automatically when its backend degrades to MP3; Soulseek falls back to queueing on busy peers and auto-relaxes its hi-res rule when nothing else is delivering; Telegram re-checks its session before every use. AI repair (optional) — SpotiFLAC is the one source that updates its own code nightly. When an update breaks it, a free built-in patch is tried first; with a key saved below, Claude then reads the error + the broken module and repairs it — sandboxed, verified, rolled back if it didn't help. Costs ~a cent on the rare day it fires.

The key is billed per use by Anthropic — there's no free tier for API calls. It only fires when a nightly update actually breaks SpotiFLAC and the free patch can't fix it, so cost is a few cents a year.

{# ── Storage paths (documentation card) ─────────────────────────── #}

Storage paths (?)

Inside the container the paths are fixed — you map them to your own folders in .env (read by docker-compose.yml), then docker compose up -d:

In container .env variable What it is
/plexify-music MUSIC_DIR Your music library — the same folder your Plex music library points at
/downloads_music DOWNLOADS_DIR Staging area for downloads (SpotiFLAC, Soulseek) before import; also holds the reversible attics
/data (fixed: ./data) Plexify's own database, logs, and rollback ledgers

Your slskd container should download into DOWNLOADS_DIR/complete so Plexify can import its files.
